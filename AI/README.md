# AI Documentation

This root directory encapsulates an orchestrated repository of AI-related skills, methodologies, prompts, and implementations targeting ML flows and generative setups. 

## How to use this folder effectively

To govern and optimize interactions with generative interfaces or programmatic ML builds, consistently enforce this workflow:

1. **Attach the Mastery Skill**: When embarking on RAG enhancements or testing architectures, enforce context bounds by loading an overarching file from [`/skills`](./skills) ensuring the model strictly adheres to pre-agreed frameworks. 
2. **Combine with Prompts**: Trigger operations using the structured macros inside [`PROMPTS.md`](./PROMPTS.md). Attaching specific domain definitions limits hallucinations and steers code output immediately correctly.
3. **Modify per Project**: Evolve boundaries efficiently. Modify localized constraints directly within `guidelines.md` to dictate varying approaches across strictly discrete subdomains (like separate rulebases for RAG implementations versus pure ML pipelines).

## Architecture & Directory Structure

To rigorously restrict context leakage when leveraging AI models, all elements exist in separated zones:

- **[`skills/`](./skills/)**: Foundational engineering plays, capability directives, and technical standards acting as rule-bounds for AI integration.
- **[`ideas/`](./ideas/)**: Scratchpad zones for feature hypothesis testing, novel prompt sequences, and experimental loops.
- **[`guidelines.md`](./guidelines.md)**: Central repository detailing consolidated stack limits, decisions, approved orchestration toolsets, and references.
- **[`PROMPTS.md`](./PROMPTS.md)**: The execution core containing copy-paste prompt blocks specifically formatted for planning, iterating, and evaluating generative work tasks flawlessly.

# Hackathon Strategy and Pitch Mastery

This mastery skill encapsulates the high-level "Product Manager" and "Architect" roles for 24-48 hour hackathons. It focuses on value-proposition, technical feasibility, and the "WOW" demo.

## Core Principles

1. **Ruthless MVP Filtering**: If a feature doesn't directly support the pitch or the "WOW" demo, it is removed from the scope.
2. **Pitch Alignment**: Every technical decision must be justifiable to a judge. Use data-driven metrics and clear analogies.
3. **Demo Resilience**: Design the system to be demo-ready early. Fake data, seed the DB, and have fallbacks ready for Wi-Fi failures.

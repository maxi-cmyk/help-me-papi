# Universal Shared Patterns

This mastery skill governs the cross-functional standards applicable to every domain in the repository (Frontend, Backend, Hardware, AI).

## Core Principles

1. **System Consistency**: Patterns must be applied uniformly to maintain a cohesive codebase.
2. **Modular Integrity**: Shared logic should be highly decoupled and strictly typed.
3. **Recursive Review**: All contributions must pass the global review checklists before integration.

---

## Navigating the Shared Domains

For any task that spans multiple engineering areas, manually attach these context files to your AI prompt:

### 1. [Universal Coding Patterns](./patterns.md)
**When to attach**: For infrastructure-level logic, shared utilities, and defining global types used across the stack.

### 2. [Cross-Domain Review Checklists](./review-checklists.md)
**When to attach**: Before merging any major PR. Ensures consistent performance, security, and stylistic standards across the entire project.

---

## Execution Workflow

1. Identify the shared pattern involved (e.g., "Generic Error Handling").
2. Attach the relevant skill or checklist from above.
3. Apply the pattern consistently to all affected domains.

# API Design Resources

## REST
- https://restfulapi.net  REST conventions reference
- https://httpstatuses.com  every HTTP status code explained
- https://developer.mozilla.org/en-US/docs/Web/HTTP/Methods  HTTP methods reference

## TypeScript validation
- https://zod.dev  Zod docs
- https://github.com/colinhacks/zod  Zod GitHub (lots of examples in README)

## tRPC
- https://trpc.io/docs  tRPC docs
- https://trpc.io/docs/quickstart  quickstart with Next.js

## GraphQL (if needed)
- https://graphql.org/learn  GraphQL basics
- https://the-guild.dev/graphql/yoga-server  GraphQL Yoga (Node.js server)
- https://pothos-graphql.dev  Pothos (code-first GraphQL with TypeScript)

## API frameworks
- https://expressjs.com  Express
- https://hono.dev  Hono (fast, edge-compatible, great TypeScript support)
- https://fastify.dev  Fastify (faster than Express, good for high-throughput APIs)
- https://nextjs.org/docs/app/building-your-application/routing/route-handlers  Next.js Route Handlers

## Rate limiting
- https://github.com/express-rate-limit/express-rate-limit  express-rate-limit
- https://upstash.com/docs/redis/sdks/ratelimit-ts/overview  Upstash rate limit (Redis-backed)

## Design references
- https://stripe.com/docs/api  Stripe API (the gold standard for REST API design)
- https://docs.github.com/en/rest  GitHub REST API (good reference for pagination, errors)

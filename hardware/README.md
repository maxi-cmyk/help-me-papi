# Hardware

This directory contains hardware-specific skills, resources, ideas, projects, and reference sheets.

## Architecture & Documentation Structure

The hardware documentation is structured into core pillars to prevent context fragmentation:

- **[`skills/SKILLS.md`](./skills/SKILLS.md)**: The foundational constraint framework and self-improvement instructions for AI interaction in hardware contexts. Read this or provide it as a system prompt to an AI before proposing any hardware changes.
- **[`inventory.md`](./inventory.md)**: User profile mapping existing platforms, confirmed used components, libraries, and a record of completed hardware projects.
- **[`optimisation-and-debugging.md`](./optimisation-and-debugging.md)**: Hardware optimisation reminders and a crucial debugging checklist. 
- **[`concepts-and-math.md`](./concepts-and-math.md)**: Quick reference sheet for bitmask logic, hardware specific math (like Shannon entropy), and structural patterns.
- **[`components.md`](./components.md)**: A catalog of anticipated components with analogies to familiar components for easy learning.
- **[`ideas/project_ideas.md`](./ideas/project_ideas.md)**: Scrappy thoughts, feature notes, and project experiments ordered by relevance to the existing skill set.
